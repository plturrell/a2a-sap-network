Design Principles

Clarity

Interfaces should be clear and self-explanatory, reducing cognitive load.

Consistency

Maintain consistent patterns across all applications and touchpoints.

Efficiency

Enable users to complete tasks with minimal effort and maximum speed.

Accessibility

Design for all users, regardless of ability or context.

Usage Guidelines

  • Use brand color for primary actions and key interactive elements
  • Apply semantic colors consistently for status indicators
  • Maintain WCAG AA contrast ratios (4.5:1 for normal text, 3:1 for large text)
  • Use color with additional indicators (icons, text) for accessibility

Accessibility

Color Contrast

Pass WCAG AA 4.5:1 ratio
Fail WCAG AA 2.1:1 ratio

Focus Indicators

Screen Reader Support

  • All interactive elements have descriptive labels
  • Form fields connected to labels via id/for attributes
  • Status messages announced via ARIA live regions
  • Logical heading hierarchy maintained
  • Alternative text for all informative images

Keyboard Navigation

Key Action Context
Tab Navigate forward All contexts
Shift + Tab Navigate backward All contexts
Enter Activate element Buttons, links
Space Toggle selection Checkboxes, buttons
Arrow keys Navigate options Lists, menus, grids
